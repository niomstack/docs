## What you'll need

<Info>
  **Before you start:**
  - **macOS 12+**, **Windows 10/11**, or **Linux** (Ubuntu 22.04+, Fedora 38+)
  - An API key from any supported provider — or [Ollama](https://ollama.ai) for fully offline use
</Info>

## Install and run

<Steps>
<Step title="Download NIOM">

  <Tabs>
    <Tab title="macOS">
      Download the `.dmg` from [GitHub Releases](https://github.com/niomstack/niom/releases).

      Open the DMG and drag NIOM to your Applications folder.

      <Tip>NIOM supports both Intel (x64) and Apple Silicon (arm64) Macs.</Tip>
    </Tab>
    <Tab title="Windows">
      Download the `.exe` installer from [GitHub Releases](https://github.com/niomstack/niom/releases).

      Run the installer — it handles everything. NIOM will appear in your system tray.
    </Tab>
    <Tab title="Linux">
      Download from [GitHub Releases](https://github.com/niomstack/niom/releases):

      ```bash
      # Debian/Ubuntu
      sudo dpkg -i niom_*.deb

      # Fedora/RHEL
      sudo rpm -i niom_*.rpm
      ```
    </Tab>
    <Tab title="From Source">
      ```bash
      git clone https://github.com/niomstack/niom.git
      cd niom
      npm install
      npm start
      ```

      **Requirements:** Node.js 22+, npm 10+
    </Tab>
  </Tabs>
</Step>

<Step title="First launch — the onboarding wizard">
  When you launch NIOM for the first time, a **4-step onboarding wizard** guides you through setup:

  1. **Welcome** — Quick overview of what NIOM can do
  2. **Connect a provider** — Enter your API key inline (Anthropic, OpenAI, or Google)
  3. **Feature tour** — Learn about memory, tasks, routing, and the `⌥ Space` hotkey
  4. **Ready** — Starter prompt suggestions to try

  <Tip>
    You can skip the API key step during onboarding and add it later via Settings. If you're running Ollama locally, NIOM detects it automatically — no key needed.
  </Tip>
</Step>

<Step title="Start chatting">
  After onboarding, you land on the **Home screen**. Type your first message in the prompt box.

  Here are some things to try:

  | Try saying... | What happens |
  |:-------------|:------------|
  | "What files are in ~/projects?" | Lists your directory structure |
  | "Search for the latest AI breakthroughs" | Runs a web search and summarizes results |
  | "Remember that I prefer dark mode and TypeScript" | Saves this to persistent memory |
  | "Help me refactor this React component" | Reads your code, suggests improvements, writes changes |
  | "Research competitor pricing and write a report" | Creates a background task with checkpoints |

  NIOM's [Skill Tree](/guides/skill-routing) analyzes your request, selects the right tools (from 20+ available), and streams a response.
</Step>
</Steps>

## The global hotkey

Press **`⌥ Space`** (Option + Space) from **any application** to summon NIOM instantly. It appears as a floating window — type your question, get your answer, and dismiss.

This is where the magic happens. NIOM becomes your ambient AI layer — always one keystroke away.

<Tip>
  NIOM also lives in your **system tray** (menubar on macOS). Right-click the tray icon for quick actions.
</Tip>

## Your data

Everything NIOM stores lives in one place:

```
~/.niom/
├── config.json       # Your settings (providers, model, theme)
├── keys.enc          # API keys (encrypted via OS keychain)
├── threads/          # Conversation history
├── context/          # Memory — facts, preferences, project data
│   ├── user/         # What NIOM knows about you
│   └── projects/     # Workspace analysis (tech stacks, conventions)
├── tasks/            # Background task state and history
├── logs/             # Tool execution logs
└── skill-graph.json  # Learned tool co-occurrence patterns
```

<Note>
  API keys are encrypted using your operating system's secure storage (macOS Keychain, Windows DPAPI, Linux Secret Service). They never leave your machine in plaintext.
</Note>
