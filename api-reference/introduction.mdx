The NIOM sidecar exposes a REST + SSE API on `localhost:3001`. The React overlay communicates with it via HTTP, and you can use it directly for automation and integration.

## Base URL

```
http://localhost:3001
```

## Authentication

No authentication required — the sidecar only binds to `localhost`.

## Key concepts

### SSE streaming

NIOM uses two SSE protocols:
- **Conversations** (`POST /run`) — AI SDK UIMessageStream with text deltas, tool calls, and reasoning
- **Tasks** (`GET /tasks/events`) — custom events: `task:start`, `task:tool`, `task:complete`, `task:error`

Both are real-time. The frontend builds state directly from SSE — no polling.

### Cascade cleanup

Deleting a thread (`DELETE /threads/:id`) cascades:
1. Deletes linked background tasks
2. Cleans up non-workspace artifacts (`~/.niom/artifacts/`)
3. Preserves workspace artifacts (your project files are never deleted)

### Artifacts

Files created by the agent are auto-registered as artifacts. Query them with `GET /artifacts?contextType=conversation&contextId={id}`. See [Artifacts guide](/guides/artifacts).
