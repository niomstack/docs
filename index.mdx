NIOM is a desktop AI agent that runs locally on your machine. It understands what you need, picks the right tools, and gets work done — from quick questions to multi-day research projects.

It's not a chatbot. It's a system.

<Note>
  NIOM is in active development (alpha). The architecture is stable and new features ship weekly.
</Note>

## What makes NIOM different

<CardGroup cols={2}>
  <Card title="Skill Tree routing" icon="sitemap">
    Every request flows through a Skill Tree — a graph of tools organized by domain (code, research, system, creative). NIOM picks the right 4-8 tools for your task, not all 15+. Faster. Cheaper. More accurate.
  </Card>
  <Card title="Tasks stream live" icon="signal">
    Background tasks don't disappear into a black box. Every tool call, every phase, every result streams to you via SSE — the same way conversations do. You see the work happening.
  </Card>
  <Card title="Artifacts you can find" icon="file-lines">
    When NIOM creates files — articles, reports, code — they're linked to the conversation or task that made them. Find them in the UI or at `~/.niom/artifacts/`. Delete a conversation? Non-workspace artifacts clean up automatically.
  </Card>
  <Card title="Deep research mode" icon="magnifying-glass">
    Multi-source research across the web. NIOM fetches, reads, cross-references, and synthesizes — with source citations. Results are saved as artifacts you can always retrieve.
  </Card>
  <Card title="Schedule detection" icon="calendar">
    Say "research AI news every morning" and NIOM creates a recurring background task automatically. No manual scheduling UI. The Skill Tree detects temporal intent and routes accordingly.
  </Card>
  <Card title="Steerable tasks" icon="compass">
    Post a comment to any running task. "Focus more on cost analysis" — the next run picks it up. Steering replaces the old approval model. You guide, NIOM executes.
  </Card>
  <Card title="Multi-model intelligence" icon="arrows-split-up-and-left">
    Fast model for analysis (~200ms). Capable model for execution. Vision model for screenshots. NIOM picks the right one per phase. You don't manage this.
  </Card>
  <Card title="Your data stays local" icon="lock">
    Everything lives at `~/.niom/`. Encrypted with AES-256-GCM. Zero telemetry. You pick your LLM provider — or run fully offline with Ollama.
  </Card>
</CardGroup>

## Under the hood

```
Your request → Skill Tree routes → Right tools selected → Agent executes → Results stream live
```

NIOM runs as two lightweight processes:

1. **Rust (Tauri)** — Thin OS shell (~200 lines). System tray, hotkeys, notifications.
2. **Node.js Sidecar** — The brain. AI SDK, Skill Tree, agent engine, tools, streaming.

Press `Ctrl+Space` to summon the overlay. It works on macOS, Windows, and Linux.
