title: "Long-Term Memory"
description: "NIOM remembers facts about you across sessions — your preferences, your projects, your patterns. It gets smarter the more you use it."
---

NIOM learns about you over time. As you chat, it extracts key facts — your name, preferred languages, current projects — and stores them encrypted in `~/.niom/memory/brain/`. These facts are injected into every conversation and influence how the [Skill Tree](/guides/skill-tree) routes your requests.

## How it works

```
You chat normally
  → Every ~6 messages, NIOM auto-extracts facts
    → Facts stored encrypted in brain/knowledge.enc
      → Next conversation loads brain context into system prompt
        → Skill Tree uses memory to refine domain routing
```

The extraction happens **asynchronously** after a thread is saved — it doesn't slow down your conversation. NIOM uses the fast model (e.g., Groq Llama 3.3) to identify facts, keeping costs negligible.

## What NIOM remembers

NIOM's brain stores three types of information:

| Type | Examples | How it learns |
|:-----|:---------|:-------------|
| **Facts** | "User's name is Arka", "Works on the NIOM project" | Auto-extracted from conversations |
| **Preferences** | `language → TypeScript`, `style → concise` | Auto-extracted or manually set |
| **Patterns** | Recurring workflows, common requests | Auto-detected over time |

### Fact extraction intelligence

The extraction system doesn't just dump everything — it's selective:
- **Merges duplicates** — if you mention your name twice in different conversations, it stores one fact, not two
- **Prioritizes actionable info** — preferences that affect tool selection, project context, tech stack
- **Skips ephemeral context** — doesn't memorize one-off questions or transient details
- **Uses structured output** — Zod schema ensures clean, parseable facts every time

## How memory influences the Skill Tree

Brain context feeds into the [Skill Tree](/guides/skill-tree) in two ways:

1. **System prompt enrichment** — every agent invocation includes your facts, so the model knows your tech stack, preferences, and context before you say anything
2. **Domain affinity** — if your memory shows you primarily write TypeScript and work on web projects, the Skill Tree subtly weights the Code domain higher for ambiguous requests

This means NIOM gets more accurate over time without any explicit training — it's just paying attention to who you are.

## Managing your memory

### Settings UI

Open **Settings → Memory** to:
- View all stored facts and preferences
- Add facts manually (e.g., "I prefer dark mode in all apps")
- Add preferences (key → value pairs)
- Remove individual facts
- Clear all brain memory

### API

You can also manage memory programmatically:

<CodeGroup>
```bash View brain
curl http://localhost:3001/memory/brain
```

```bash Add a fact
curl -X POST http://localhost:3001/memory/brain/fact \
  -H "Content-Type: application/json" \
  -d '{"fact": "I prefer TypeScript over JavaScript"}'
```

```bash Set a preference
curl -X POST http://localhost:3001/memory/brain/pref \
  -H "Content-Type: application/json" \
  -d '{"key": "editor", "value": "VS Code"}'
```

```bash Remove a fact
curl -X DELETE http://localhost:3001/memory/brain/fact \
  -H "Content-Type: application/json" \
  -d '{"fact": "I prefer TypeScript over JavaScript"}'
```

```bash Clear all memory
curl -X DELETE http://localhost:3001/memory/brain
```
</CodeGroup>

## Task memory vs brain memory

NIOM has two memory systems — don't confuse them:

| | Brain memory | Task memory |
|:--|:------------|:------------|
| **Scope** | Global — applies to all conversations | Per-task — scoped to one task |
| **What it stores** | Facts, preferences, patterns about you | Run history, steering comments, tool results |
| **How it learns** | Auto-extracted from conversations | Built during task execution |
| **Where it lives** | `~/.niom/memory/brain/knowledge.enc` | `~/.niom/memory/runs/{task-id}/` |
| **Used by** | Every conversation + Skill Tree | Only that specific task's future runs |

Both are encrypted with AES-256-GCM.

## Privacy

<Note>
  Brain data is **encrypted at rest** with AES-256-GCM, just like conversations and tasks. It never leaves your machine. Facts are only extracted from your local conversations — nothing is sent to any external service beyond your chosen LLM provider.
</Note>

## How it improves NIOM

With brain context, NIOM can:

- **Skip repetitive questions** — it already knows your name, stack, and preferences
- **Tailor responses** — if it knows you use TypeScript, it won't suggest Python solutions
- **Build on past context** — "continue working on the NIOM project" just works
- **Personalize tone** — matching your preferred communication style over time
- **Route more accurately** — the [Skill Tree](/guides/skill-tree) uses your profile to resolve ambiguous requests

<Tip>
  Memory is most useful after 3-5 conversations. That's enough for NIOM to build a solid profile of who you are and how you work.
</Tip>
