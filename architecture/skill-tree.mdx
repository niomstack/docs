The Skill Tree is NIOM's intelligent routing system. It's a **directed acyclic graph (DAG)** that selects the right tools for every user request — without making a single LLM call.

> Based on the research paper: *"Skill Trees for AI Agents: Hierarchical Tool Selection via Self-Improving Graphs"* (Singh, Bhargavi, Singh — Obelysk, 2026)

## The problem

As tool inventories grow, LLM tool selection accuracy degrades sharply:

| Tool count | Accuracy |
|:-----------|:---------|
| 5 tools | ~95% |
| 15 tools | ~82% |
| 30 tools | ~68% |
| 50+ tools | Sharp degradation |

Most AI apps either dump all tools into the context (degrading quality) or hard-code routing rules (no adaptability). NIOM's Skill Tree solves both.

## Data structures

### SkillNode

Represents a domain or tool in the graph. Carries:
- Embeddings and exemplar embeddings
- Usage statistics (invocations, last used)
- Parent/child relationships
- Pack membership

Types: `root` → `domain` → `subdomain` → `tool`

### SkillEdge

Weighted, typed edges between nodes:

| Edge type | Created by | Evolves by |
|:----------|:-----------|:-----------|
| `hierarchy` | Install-time (pack structure) | Never decayed — structural |
| `semantic` | Install-time (cosine sim > 0.6) | Decayed if unused |
| `cooccurrence` | Runtime (tools used together) | Reinforced on co-use, decayed over time |
| `pipeline` | Runtime (tool B follows tool A) | Reinforced on sequential use |

### SkillPath

The output of traversal — what gets injected into the agent:

```typescript
{
  tools: Tool[],              // Selected tool subset (4-8 typical)
  primaryDomain: string,      // Top-scoring domain
  secondaryDomains: string[], // Supporting domains
  confidence: number,         // Routing confidence (0-1)
  systemPromptFragments: string[], // Domain-specific personality
  stepBudget: number          // Max agent steps for this query type
}
```

## Three-phase routing pipeline

| Phase | When | Latency | What happens |
|:------|:-----|:--------|:-------------|
| **A: Install-time** | First launch | ~8s (then cached) | Parse packs, embed exemplars, build DAG, persist |
| **B: Type-time** | On user input | **~2ms** | Embed query, hierarchical traversal, resolve SkillPath |
| **C: Send-time** | On send | ~0ms | Inject cached SkillPath into agent `prepareCall` |

## Exemplar embeddings

The key insight: standard embedding models produce near-zero cosine similarity between short user queries and long tool descriptions. NIOM uses **two-level exemplar embeddings** to fix this:

### Domain exemplars (5-13 per domain)
Short phrases that capture when a domain should activate:
- "read this file"
- "run the build command"
- "search for information"

### Tool exemplars (2-4 per tool)
Usage-oriented phrases:
- "save this report to a file"
- "create a new document with this content"

Scoring uses **max-similarity** — `max(cos(query, exemplar_i))` — so ANY exemplar matching activates the domain/tool.

## Hierarchical traversal algorithm

```
1. Embed user query
2. Score all domains: max-sim(query, domain_exemplars) + usage_boost
3. Select top-3 domains + OS (guaranteed) + Web (if research/business/creative)
4. Score tools within selected domains: cosine_sim + domain_boost + cooccurrence_bonus
5. Apply tiered adaptive cutoff:
   - Primitive tools: relaxed cutoff (mean)
   - Intent tools: strict cutoff (mean + 0.25 * stddev)
   - Clamp: 3-6 tools per domain
6. Build SkillPath, cache with LRU (30s TTL)
```

Total comparisons: ~98 per query (vs ~19 for flat). The ~2ms overhead is imperceptible.

## Temporal EMA edge learning

After each tool execution, NIOM updates edges:

```
w(t+1) = w(t) · exp(-λ · dt) + α · (1 - w(t) · exp(-λ · dt))
```

Where:
- `α = 0.15` — learning rate (diminishing returns)
- `λ = ln(2) / 30 days` — 30-day half-life
- `dt` — time since last reinforcement

Properties:
- **Fast learning** — 0 to 0.15 on first reinforcement
- **Gradual decay** — 50% strength after 30 days unused
- **Asymptotic** — approaches 1.0 but never reaches it
- **Automatic** — no manual intervention needed

## Benchmark results

On a 300-query benchmark across 6 domains:

| Metric | Flat embedding | Skill Tree | LLM (Claude Opus) |
|:-------|:--------------|:-----------|:-------------------|
| **F1** | 0.332 | **0.384** (+15.7%) | 0.661 |
| **Recall** | 0.714 | **0.660** | 0.650 |
| **Latency** | 0.03ms | **2.0ms** | 1,863ms |
| **LLM calls** | 0 | **0** | 300 |
| **Cost** | $0.00 | **$0.00** | $0.257 |

The Skill Tree achieves **recall parity with Claude Opus** while being **1,025x faster** and consuming **zero tokens.**
