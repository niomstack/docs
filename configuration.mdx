NIOM is configured through the **Settings view** in the app, or by editing `~/.niom/config.json` directly. API keys are stored separately in `~/.niom/keys.enc` (encrypted).

## Providers

NIOM supports four model providers. You can configure multiple providers and switch between them anytime.

| Provider | Models | Best for |
|:---------|:-------|:---------|
| **Anthropic** | Claude 4, Claude Sonnet | Complex reasoning, tool use, coding |
| **OpenAI** | GPT-4.5, GPT-4o | General use, vision, versatility |
| **Google** | Gemini 2.5 Pro/Flash | Large context windows, multimodal |
| **Ollama** | Llama 3, Mistral, Phi, etc. | Full privacy — runs locally, no API key |

### Adding a provider

<Tabs>
  <Tab title="In the app">
    Open **Settings** → scroll to **Providers** → enter your API key → click **Save**.

    The provider is immediately enabled. Switch your default model in the model selector at the bottom of the chat.
  </Tab>
  <Tab title="During onboarding">
    The onboarding wizard on first launch has an inline API key entry step. Add one or more keys and you're set.
  </Tab>
  <Tab title="Config file">
    Edit `~/.niom/config.json`:

    ```json
    {
      "providers": [
        { "id": "anthropic", "name": "Anthropic", "enabled": true },
        { "id": "openai", "name": "OpenAI", "enabled": true },
        { "id": "google", "name": "Google", "enabled": false },
        { "id": "ollama", "name": "Ollama (Local)", "enabled": false, "baseUrl": "http://localhost:11434" }
      ],
      "defaultModel": "anthropic:claude-sonnet-4-20250514",
      "ollamaUrl": "http://localhost:11434",
      "theme": "dark"
    }
    ```

    API keys are stored separately in `~/.niom/keys.enc` and managed through the Settings UI. They're encrypted via your OS keychain.
  </Tab>
</Tabs>

## Default model

The default model is used for all new conversations. You can override it per-thread using the model selector.

The format is `provider:model-id`, for example:

| Model ID | Provider | Notes |
|:---------|:---------|:------|
| `anthropic:claude-sonnet-4-20250514` | Anthropic | **Recommended** — best balance of speed + capability |
| `anthropic:claude-4-20250514` | Anthropic | Most capable, slower |
| `openai:gpt-4.5-preview` | OpenAI | Strong general purpose |
| `openai:gpt-4o` | OpenAI | Fast, multimodal |
| `google:gemini-2.5-pro` | Google | Huge context window |
| `ollama:llama3.1` | Ollama | Fully offline |

## Ollama (local models)

NIOM auto-detects Ollama at `http://localhost:11434`. No API key needed — just install Ollama and pull a model:

```bash
# Install Ollama
curl -fsSL https://ollama.ai/install.sh | sh

# Pull a model
ollama pull llama3.1

# NIOM auto-detects it — no config needed
```

If Ollama is running on a different port or machine, set the URL in Settings or in `config.json`:

```json
{
  "ollamaUrl": "http://192.168.1.100:11434"
}
```

<Tip>
  For the best experience with local models, we recommend at least **8GB VRAM** and a model with 7B+ parameters. Smaller models work but may struggle with complex tool use.
</Tip>

## Theme

NIOM supports dark and light mode. Set it in Settings or in `config.json`:

```json
{
  "theme": "dark"
}
```

The theme applies immediately — no restart needed.

## File locations

| File | Purpose |
|:-----|:--------|
| `~/.niom/config.json` | Main configuration — providers, default model, theme, Ollama URL |
| `~/.niom/keys.enc` | Encrypted API keys (OS keychain) |
| `~/.niom/threads/` | Conversation history (JSON per thread) |
| `~/.niom/context/user/` | Your persistent memory — facts, preferences |
| `~/.niom/context/projects/` | Auto-detected workspace analysis |
| `~/.niom/tasks/` | Background task state and history |
| `~/.niom/skill-graph.json` | Learned tool usage patterns |
| `~/.niom/logs/` | Tool execution audit trail |

<Note>
  All data is portable. Copy `~/.niom/` to a new machine and everything transfers — conversations, memories, settings, and learned patterns.
</Note>
