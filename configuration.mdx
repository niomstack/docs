NIOM keeps things simple: **one configuration file** at `~/.niom/config.json`. No `.env` files, no environment variables to juggle.

## Common setups

<AccordionGroup>
  <Accordion title="Minimal (cloud AI, zero extras)">
    The bare minimum to get running. Uses Claude with DuckDuckGo search (no API keys needed for search):

    ```json
    {
      "workspace": "C:\\Users\\you",
      "provider": "anthropic",
      "model": "anthropic/claude-4-sonnet-20250514",
      "anthropic_api_key": "sk-ant-..."
    }
    ```
  </Accordion>
  <Accordion title="Power user (multi-model + premium search)">
    Fast analysis with Groq, main work with Claude, vision with GPT-4o, upgraded search:

    ```json
    {
      "workspace": "C:\\Users\\you",
      "provider": "anthropic",
      "model": "anthropic/claude-4-sonnet-20250514",
      "anthropic_api_key": "sk-ant-...",
      "groq_api_key": "gsk_...",
      "openai_api_key": "sk-...",
      "search": { "provider": "tavily", "api_key": "tvly-..." },
      "models": {
        "fast": "groq/llama-3.3-70b-versatile",
        "vision": "openai/gpt-4o"
      },
      "mcp": [
        {
          "name": "github",
          "command": "github-mcp-server",
          "env": { "GITHUB_TOKEN": "ghp_..." }
        }
      ]
    }
    ```
  </Accordion>
  <Accordion title="Fully offline (Ollama, zero network)">
    Everything runs locally. No API keys, no cloud, no network calls:

    ```json
    {
      "workspace": "C:\\Users\\you",

        "fast": "ollama/llama3.1",

      }
    }
    ```

    <Tip>Make sure [Ollama](https://ollama.ai) is installed and running locally.</Tip>

## Full reference

### Core settings

| Field | Type | Default | What it does |
|:------|:-----|:--------|:------------|
| `workspace` | string | Home directory | Root directory for file operations — NIOM works within this folder |
| `provider` | string | `"anthropic"` | Which LLM provider to use: `anthropic`, `openai`, `google`, `groq`, `ollama` |
| `model` | string | `"anthropic/claude-4-sonnet-20250514"` | The main model for execution |
| `sidecar_port` | number | `3001` | Port the AI sidecar listens on |

### Provider API keys

Configure keys directly per provider. Only the key for your selected provider is required:

| Field | Provider |
|:------|:---------|
| `anthropic_api_key` | Anthropic (Claude) |
| `openai_api_key` | OpenAI (GPT-4o, o1, etc.) |
| `google_api_key` | Google (Gemini) |
| `groq_api_key` | Groq (Llama, Mixtral) |

<Tip>Ollama doesn't need an API key — just install it and set `provider` to `"ollama"`.</Tip>

### Search

NIOM searches the web automatically when needed. Three providers are supported, chained with automatic fallback:

| Field | Type | Default | What it does |
|:------|:-----|:--------|:------------|
| `search.provider` | `"duckduckgo"` `"tavily"` `"brave"` | `"duckduckgo"` | DuckDuckGo works without a key. Tavily and Brave are faster and richer. |
| `search.api_key` | string | — | API key for Tavily ([free: 1,000/month](https://tavily.com)) or Brave ([free: 2,000/month](https://brave.com/search/api/)) |

**Provider chain:** If your configured provider fails, NIOM falls back to DuckDuckGo automatically. Zero-config search always works.

### Model roles

NIOM uses different models for different jobs — so you're not paying premium prices for simple analysis:

| Field | Used for | Default | Why |
|:------|:---------|:--------|:----|
| `model` | Main execution | Claude Sonnet | The workhorse. Handles tool calls, reasoning, and responses |
| `models.fast` | Analysis + routing + evaluation | Groq Llama 3.3 70B | Sub-second, cheap. Used for Skill Tree routing and quality checks |
| `models.vision` | Screenshot understanding | GPT-4o | Needed for computer use — reads what's on your screen |

### MCP servers

Connect external tools by adding MCP servers. They auto-start when the sidecar boots:

```json
{
  "mcp": [
    {
      "name": "github",
      "command": "github-mcp-server",
      "args": [],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    }
  ]
}
```

<Tip>
  See the [MCP Integration guide](/guides/mcp) for setup instructions and a list of popular servers.
</Tip>

## Where your data lives

```
~/.niom/
├── config.json              # This file — all your settings
├── memory/                  # Encrypted persistence layer
│   ├── index.json           # Fast lookup index
│   ├── conversations/       # Chat threads (AES-256-GCM)
│   ├── tasks/               # Background task definitions
│   ├── runs/                # Task execution history
│   │   └── {task-id}/
│   │       └── run_NNNN.enc
│   └── brain/               # Long-term memory
│       └── knowledge.enc    # Facts, preferences, patterns
├── artifacts/               # Files generated by NIOM
│   └── {context-id}/        # Scoped by conversation or task
│       ├── manifest.json    # What was generated + metadata
│       └── article.md       # The actual file
├── skill-graph.json         # Skill Tree co-occurrence data
├── screenshots/             # Computer use (last 5 kept)
└── mcp/                     # MCP configs (future)
```

<Note>
  All data in `memory/` is encrypted with AES-256-GCM. Even if someone accesses your `~/.niom/` directory, they can't read your conversations, tasks, or learned facts.
</Note>

<Note>
  **Artifacts** are stored at `~/.niom/artifacts/` by default. If your conversation has a workspace connected, NIOM saves artifacts there instead — inside your project where they belong. See [Artifacts](/guides/artifacts).
</Note>

## How to update your config

Three ways, pick whichever suits you:

1. **Settings UI** — open the overlay → Settings. Provider selection, API key management, model picker.
2. **Edit the file directly** — change `~/.niom/config.json` and restart the sidecar.
3. **API call** — `POST /providers/configure` ([docs](/api-reference/config/providers))
