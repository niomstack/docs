Most AI tools run a simple loop: get your prompt, call an LLM, return text. If you're lucky, it uses a tool.

**NIOM is different.** Every message flows through a [Skill Tree](/guides/skill-tree) that picks the right tools, then through a reasoning pipeline that adapts effort based on complexity — from instant responses to multi-step execution with quality checks.

## How it works

```
Your message → Skill Tree routes → Tools selected → Execute → Evaluate → Respond (or refine)
```

<Steps>
<Step title="1. Skill Tree routing">
  The [Skill Tree](/guides/skill-tree) classifies your message into a domain (code, research, creative, etc.) and selects 4-8 focused tools. This takes ~200ms using the fast model.

  Simple messages like "hi" skip routing entirely — zero added latency.
</Step>
<Step title="2. Complexity assessment">
  Based on the Skill Tree's analysis, NIOM chooses how much effort to invest:

  | Complexity | Example | What NIOM does |
  |:----------|:--------|:--------------|
  | **Simple** | "What's my CPU usage?" | Quick response, 3 tool calls max |
  | **Standard** | "List my project structure" | Executes with light evaluation |
  | **Complex** | "Refactor this module" | Full pipeline with quality checks |
  | **Long-running** | "Research AI trends every week" | Creates a [background task](/guides/background-tasks) |
</Step>
<Step title="3. Execute with tools">
  The agent works with only the tools the Skill Tree selected — reading files, running commands, searching the web, taking screenshots, or calling MCP tools.
</Step>
<Step title="4. Evaluate quality">
  After execution, a separate evaluation step asks: Did we achieve the goal? What's the quality score?

  For complex tasks, this creates a refinement loop: execute → evaluate → improve → re-execute (up to 3 rounds) until the quality bar is met.
</Step>
</Steps>

## Smart model routing

NIOM uses the right model for each phase — so you're not paying premium prices for simple work:

| Role | What it does | Default | Why |
|:-----|:------------|:--------|:----|
| **Fast** | Skill Tree routing, intent analysis, quality evaluation | Groq Llama 3.3 70B | Sub-second, nearly free |
| **Capable** | Main execution — the actual work | Your selected model | The workhorse |
| **Vision** | Screenshot understanding | GPT-4o or Claude | Needed for computer use |

Analysis happens in ~200ms on a free-tier model. Your premium model is reserved for the work that matters.

## Why this architecture

| Traditional agents | NIOM |
|:-------------------|:-----|
| All tools loaded for every request | Skill Tree selects 4-8 relevant tools |
| LLM decides when to stop | Quality criteria defined upfront |
| No quality assessment | Explicit evaluation with refinement loops |
| Same effort for "hi" and "refactor my codebase" | Adaptive depth based on complexity |
| Can't loop back to improve | Execute → evaluate → refine → re-execute |
| One model for everything | Right model per phase = faster + cheaper |

<Tip>
  This pipeline is what makes [background tasks](/guides/background-tasks), [deep research](/guides/deep-research), and [computer use](/guides/computer-use) reliable — every tool call goes through structured reasoning, not hope.
</Tip>
