Background Tasks are NIOM's superpower for complex, multi-step work. Instead of waiting for a single long response, you dispatch a task and keep chatting while NIOM works autonomously.

## What is a Task?

A Task is a **multi-phase, pausable, background workflow**. Think of it as giving NIOM a mission.

| | Regular chat | Background task |
|:---|:------------|:----------------|
| **Duration** | Seconds to minutes | Minutes to hours |
| **Phases** | Single response | Plan → Execute → Synthesize |
| **Interaction** | Foreground (streaming) | Background (progress updates) |
| **Interruption** | Cancel | Pause / resume / steer / checkpoint |
| **Result** | Streamed text | Structured deliverable |

## Creating a task

### From conversation

Type a complex request naturally. If NIOM detects it's better suited as a background task, it suggests:

> _"This looks like it could take a while. Run as a background Task?"_

Detection signals include:
- Multi-step goals ("research X, then compare Y, then write Z")
- Temporal markers ("over the next week", "every day")
- Cross-domain work (research + writing + code)
- Keywords like "deeply", "comprehensive", "full report"

### Explicit mode

Click the **⚡ Task toggle** next to the send button in the prompt box. When active, your next message dispatches as a Task instead of a regular chat message.

You can also use task templates — pre-built workflows for common scenarios like research reports, code reviews, and competitive analysis.

## The three phases

### Phase 1: Plan

NIOM decomposes your goal into 3-7 concrete steps. You see the plan before any work begins:

```
📋 Plan for "Research AI agent frameworks and compare them"

Step 1: Search for popular AI agent frameworks in 2026
Step 2: Deep-dive into the top 5 candidates
Step 3: Compare features, pricing, and architecture
Step 4: Synthesize findings into a comparison report

Estimated: ~4 minutes | ~2K tokens
```

You can **approve**, **modify**, or **reject** the plan. Nothing executes until you say go.

### Phase 2: Execute

Each step runs as an independent agent call with full access to NIOM's tools and Skill Tree routing. Progress streams in real-time:

```
🔄 Step 1/4: Searching for AI agent frameworks...
  ✅ webSearch: "AI agent frameworks 2026" → 12 results
  ✅ crawl: langchain.dev → extracted 2.3K words
  ✅ crawl: crewai.com → extracted 1.8K words
✅ Step 1 complete

🔄 Step 2/4: Deep-diving top candidates...
```

### Phase 3: Synthesize

After all steps complete, NIOM produces a final deliverable — a report, summary, code, or whatever the task calls for. The result appears as a rich message in your thread.

## Checkpoints

Tasks can **pause and ask for your input** at critical moments:

### Mandatory checkpoints
- **After planning** — You always review the plan before execution
- **On error** — When a step fails and can't self-recover

### Configurable checkpoints
- **After every N steps** — Default: after every step (conservative). Adjust via task templates.
- **On significant findings** — The agent can flag unexpected or contradictory results

When a checkpoint appears, you see a card in the thread with three options:

- **Continue** — Proceed as planned
- **Modify direction** — Type new guidance, NIOM adjusts and continues
- **Stop here** — Cancel the remaining steps, keep what's done

## Managing tasks

### In your thread

The **Task Panel** in the prompt area shows:
- Active tasks with live progress
- Step list (✅ completed, 🔄 current, ⏳ pending)
- Quick actions: Pause / Resume / Cancel

### Global view

The **Home screen** shows all tasks across all threads:
- Filterable by status: Running | Paused | Completed | Failed
- Each card shows: goal, source thread, progress, elapsed time
- Click a task → navigate to the source thread

## Error handling

Tasks handle errors at three levels:

| Level | What | Handling |
|:------|:-----|:---------|
| **Tool-level** | A tool call fails | Automatic retry (up to 3x) via agent self-correction |
| **Step-level** | An entire step fails | Checkpoint card: Retry / Skip / Stop |
| **System-level** | API error, rate limit, network | Auto-backoff and retry. Pauses if persistent. |

## Task persistence

Tasks are **crash-safe**. They save state to `~/.niom/tasks/` after every step. If NIOM restarts:

- Completed steps are preserved
- Running steps resume from the last checkpoint
- Failed steps can be retried

<Note>
  Tasks run in the main process. They survive the app being minimized or the tray icon being idle, but not the app being fully quit.
</Note>
