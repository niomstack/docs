## What you'll need

<Info>
  **Before you start:**
  - **Windows 10/11**, **macOS 12+**, or **Linux** (Ubuntu 22.04+, Fedora 38+, Arch)
  - An API key from any AI provider — [OpenAI](https://platform.openai.com), [Anthropic](https://console.anthropic.com), or [Google](https://aistudio.google.com) all work
</Info>

## Install and run

<Steps>
<Step title="Install NIOM">

  <Tabs>
    <Tab title="macOS">
      **Homebrew:**
      ```bash
      brew tap niomstack/niom https://github.com/niomstack/niom
      brew install --cask niom
      ```

      Or download the `.dmg` from [GitHub Releases](https://github.com/niomstack/niom/releases).
    </Tab>
    <Tab title="Windows">
      **Scoop:**
      ```powershell
      scoop bucket add niom https://github.com/niomstack/niom
      scoop install niom
      ```

      Or download the `.exe` installer from [GitHub Releases](https://github.com/niomstack/niom/releases).
    </Tab>
    <Tab title="Linux">
      Download from [GitHub Releases](https://github.com/niomstack/niom/releases):

      ```bash
      # Debian/Ubuntu
      sudo dpkg -i niom_*.deb

      # Any distro (AppImage)
      chmod +x niom_*.AppImage
      ./niom_*.AppImage
      ```
    </Tab>
    <Tab title="From Source">
      ```bash
      git clone https://github.com/niomstack/niom.git
      cd niom
      pnpm install
      pnpm --filter niom-ai install
      pnpm tauri dev
      ```

      **Requirements:** Node.js 22+, Rust 1.75+, pnpm 9+
    </Tab>
  </Tabs>
</Step>

<Step title="Start NIOM">
  Launch NIOM from your Applications folder, Start Menu, or app launcher. It runs in your **system tray**.

  <Tip>Building from source? Run `pnpm tauri dev` to start all three services at once.</Tip>
</Step>

<Step title="Connect your AI provider">
  Press **Ctrl+Space** to open the overlay, then go to **Settings** and enter your API key.

  Or configure via the terminal:
  ```bash
  curl -X POST http://localhost:3001/providers/configure \
    -H "Content-Type: application/json" \
    -d '{"model": "anthropic/claude-4-sonnet-20250514", "gateway_key": "your-key-here"}'
  ```

  <Tip>
    Don't want to use cloud AI? NIOM works fully offline with [Ollama](https://ollama.ai). Just set the provider to `ollama` and choose a local model.
  </Tip>
</Step>

<Step title="Try it out">
  Press **Ctrl+Space** and type something. Here are a few ideas:

  | Try saying... | What happens |
  |---------------|-------------|
  | `"What files are in my project?"` | NIOM reads your directory structure |
  | `"Search for the latest AI research"` | Runs a web search and summarizes results |
  | `"Monitor my downloads folder"` | Creates a background task that watches for changes |
  | `"Take a screenshot and describe it"` | Captures your screen and uses vision AI to explain it |

  NIOM will analyze your request, pick the right tools, and stream a response — all in the overlay.
</Step>
</Steps>

## Make sure it's working

Check that the sidecar is healthy:

```bash
curl http://localhost:3001/health
```

You should see:
```json
{
  "status": "ok",
  "uptime_ms": 12345,
  "version": "0.1.0"
}
```

<Warning>
  If you get `connection refused`, the sidecar hasn't started yet. Wait a few seconds after `pnpm tauri dev` — it boots asynchronously.
</Warning>

## Your config file

Everything lives in one place: `~/.niom/config.json`. No `.env` files, no scattered secrets.

```json
{
  "workspace": "C:\\Users\\you",
  "gateway_key": "your-key-here",
  "provider": "anthropic",
  "model": "anthropic/claude-4-sonnet-20250514",
  "sidecar_port": 3001,
  "search": {
    "provider": "brave",
    "api_key": "BSA..."
  },
  "models": {
    "fast": "groq/llama-3.3-70b-versatile",
    "vision": "openai/gpt-4o"
  },
  "mcp": []
}
```

<Tip>
  See the [Configuration reference](/configuration) for every option explained.
</Tip>
