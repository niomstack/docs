## What you'll need

<Info>
  **Before you start:**
  - **Windows 10/11**, **macOS 12+**, or **Linux** (Ubuntu 22.04+, Fedora 38+, Arch)
  - An API key from any supported provider — [OpenAI](https://platform.openai.com), [Anthropic](https://console.anthropic.com), [Google](https://aistudio.google.com), or [Groq](https://console.groq.com)
</Info>

## Install and run

<Steps>
<Step title="Install NIOM">

  <Tabs>
    <Tab title="macOS">
      **Homebrew:**
      ```bash
      brew tap niomstack/niom https://github.com/niomstack/niom
      brew install --cask niom
      ```

      Or download the `.dmg` from [GitHub Releases](https://github.com/niomstack/niom/releases).
    </Tab>
    <Tab title="Windows">
      **Scoop:**
      ```powershell
      scoop bucket add niom https://github.com/niomstack/niom
      scoop install niom
      ```

      Or download the `.exe` installer from [GitHub Releases](https://github.com/niomstack/niom/releases).
    </Tab>
    <Tab title="Linux">
      Download from [GitHub Releases](https://github.com/niomstack/niom/releases):

      ```bash
      # Debian/Ubuntu
      sudo dpkg -i niom_*.deb

      # Any distro (AppImage)
      chmod +x niom_*.AppImage
      ./niom_*.AppImage
      ```
    </Tab>
    <Tab title="From Source">
      ```bash
      git clone https://github.com/niomstack/niom.git
      cd niom
      pnpm install
      pnpm --filter niom-ai install
      pnpm tauri dev
      ```

      **Requirements:** Node.js 22+, Rust 1.75+, pnpm 9+
    </Tab>
  </Tabs>
</Step>

<Step title="Start NIOM">
  Launch NIOM from your Applications folder, Start Menu, or app launcher. It runs in your **system tray**.

  <Tip>Building from source? Run `pnpm tauri dev` to start all three services at once.</Tip>
</Step>

<Step title="Connect your AI provider">
  Press **Ctrl+Space** to open the overlay, then go to **Settings** and select your provider + enter your API key.

  NIOM supports:
  - **Anthropic** (Claude) — recommended for best tool-use performance
  - **OpenAI** (GPT-4o, o1, etc.)
  - **Google** (Gemini)
  - **Groq** (fast, free tier available)
  - **Ollama** (fully offline, no API key needed)

  <Tip>
    Don't want to use cloud AI? NIOM works fully offline with [Ollama](https://ollama.ai). Just set the provider to `ollama` and choose a local model. No API key required.
  </Tip>
</Step>

<Step title="Try it out">
  Press **Ctrl+Space** and type something. Here are a few ideas:

  | Try saying... | What happens |
  |:-------------|:------------|
  | "What files are in my project?" | Reads your directory structure |
  | "Search for the latest AI research" | Runs a web search and summarizes results |
  | "Research quantum computing deeply" | Multi-source deep research with citations |
  | "Write a summary of AI trends every 2 days" | Creates a scheduled background task |
  | "Take a screenshot and describe it" | Captures your screen and uses vision AI |

  NIOM's [Skill Tree](/guides/skill-tree) analyzes your request, selects the right tools, and streams a response — all in the overlay.
</Step>
</Steps>

## Make sure it's working

Check that the sidecar is healthy:

```bash
curl http://localhost:3001/health
```

You should see:
```json
{
  "status": "ok",
  "uptime_ms": 12345,
  "version": "0.1.0"
}
```

<Warning>
  If you get `connection refused`, the sidecar hasn't started yet. Wait a few seconds after launch — it boots asynchronously.
</Warning>

## Your config file

Everything lives in one place: `~/.niom/config.json`. No `.env` files, no scattered secrets.

```json
{
  "workspace": "C:\\Users\\you",
  "provider": "anthropic",
  "model": "anthropic/claude-4-sonnet-20250514",
  "anthropic_api_key": "sk-ant-...",
  "sidecar_port": 3001,
  "models": {
    "fast": "groq/llama-3.3-70b-versatile",
    "vision": "openai/gpt-4o"
  }
}
```

**That's it.** Web search works out of the box (DuckDuckGo, no API key). Artifacts save to `~/.niom/artifacts/` by default. The [Skill Tree](/guides/skill-tree) builds itself as you use NIOM.

<Tip>
  See the [Configuration reference](/configuration) for every option explained.
</Tip>
